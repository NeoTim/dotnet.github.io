Interface IObservable<T>

Defines a provider for push-based notification.

Namespace:System
Assembly:System.Runtime.dll
Syntax
public interface IObservable<out T>
Remarks

The IObserver and IObservable interfaces provide a generalized mechanism for push-based notification, also known as the observer design pattern. The IObservable interface represents the class that sends notifications (the provider); the IObserver interface represents the class that receives them (the observer). T represents the class that provides the notification information. In some push-based notifications, the IObserver implementation and T can represent the same type.

The provider must implement a single method, Subscribe(IObserver), that indicates that an observer wants to receive push-based notifications. Callers to the method pass an instance of the observer. The method returns an IDisposable implementation that enables observers to cancel notifications at any time before the provider has stopped sending them.

At any given time, a given provider may have zero, one, or multiple observers. The provider is responsible for storing references to observers and ensuring that they are valid before it sends notifications. The IObservable interface does not make any assumptions about the number of observers or the order in which notifications are sent.

The provider sends the following three kinds of notifications to the observer by calling IObserver methods:

  • The current data. The provider can call the OnNext(T) method to pass the observer a T object that has current data, changed data, or fresh data.

  • An error condition. The provider can call the OnError(Exception) method to notify the observer that some error condition has occurred.

  • No further data. The provider can call the OnCompleted() method to notify the observer that it has finished sending notifications.

Subscribe(IObserver<T>)

Notifies the provider that an observer is to receive notifications.

Remarks

The Subscribe(IObserver) method must be called to register an observer for push-based notifications. A typical implementation of the Subscribe(IObserver) method does the following:

  • It stores a reference to the observer in a collection object, such as a List object.

  • It returns a reference to an IDisposable interface. This enables observers to unsubscribe (that is, to stop receiving notifications) before the provider has finished sending them and called the subscriber's OnCompleted() method.

At any given time, a particular instance of an IObservable implementation is responsible for handling all subscriptions and notifying all subscribers. Unless the documentation for a particular IObservable implementation indicates otherwise, observers should make no assumptions about the IObservable implementation, such as the order of notifications that multiple observers will receive.
