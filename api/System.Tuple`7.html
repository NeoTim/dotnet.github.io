IStructuralComparable.CompareTo(Object, IComparer)

Compares the current Tuple object to a specified object by using a specified comparer, and returns an integer that indicates whether the current object is before, after, or in the same position as the specified object in the sort order.

Remarks

This member is an explicit interface implementation. It can be used only when the Tuple instance is cast to an IStructuralComparable interface.

This method lets you define customized comparisons of Tuple objects. For example, you can use this method to order Tuple objects based on the value of a specific component.

Although this method can be called directly, it is most commonly called by collection-sorting methods that include IComparer parameters to order the members of a collection. For example, it is called by the Sort(Array, IComparer) method and the Add(Object, Object) method of a SortedList object that is instantiated by using the System.Collections.SortedList. constructor.

The System.Tuple`7.System method is intended for use in sorting operations. It should not be used when the primary purpose of a comparison is to determine whether two objects are equal. To determine whether two objects are equal, call the System.Tuple`7.System method.

The example then displays the elements in the array in unsorted order, sorts them by the third component (the population in 1960) and displays them, and then sorts them by the sixth component (the population in 1990) and displays them.

The Equals(Object, Object) implementation is called only if other is not null, and if it can be successfully cast (in C#) or converted (in Visual Basic) to a Tuple object whose components are of the same types as the current instance. The System.Tuple`7.System method first passes the Item1 values of the Tuple objects to be compared to the Equals(Object, Object) implementation. If this method call returns true, the method is called again and passed the Item2 values of the two Tuple objects.
