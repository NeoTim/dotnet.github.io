Struct ArraySegment<T>

Delimits a section of a one-dimensional array.

Namespace:System
Assembly:System.Runtime.dll
Syntax
[StructLayout(LayoutKind.Sequential)]
public struct ArraySegment<T> : IList<T>, ICollection<T>, IReadOnlyList<T>, IReadOnlyCollection<T>, IEnumerable<T>, IEnumerable
Remarks

ArraySegment is a wrapper around an array that delimits a range of elements in that array. Multiple ArraySegment instances can refer to the same original array and can overlap. The original array must be one-dimensional and must have zero-based indexing.

ArraySegment implements the IReadOnlyCollection interface starting with the .NET Framework 4.6; in previous versions of the .NET Framework, the ArraySegment structure did not implement this interface.

The ArraySegment structure is useful whenever the elements of an array will be manipulated in distinct segments. For example:

  • You can pass an ArraySegment object that represents only a portion of an array as an argument to a method, rather than call a relatively expensive method like Copy(Array, Int32, Array, Int32, Int32) to pass a copy of a portion of an array.

  • In a multithreaded app, you can use the ArraySegment structure to have each thread operate on only a portion of the array.

  • For task-based asynchronous operations, you can use an ArraySegment object to ensure that each task operates on a distinct segment of the array. The following example divides an array into individual segments with up to ten elements. Each element in the segment is multiplied by its segment number. The result shows that using the ArraySegment class to manipulate elements in this way changes the values of its underlying array.

    System.ArraySegment#2

Note, however, that although the ArraySegment structure can be used to divide an array into distinct segments, the segments are not completely independent of one another. The Array property returns the entire original array, not a copy of the array; therefore, changes made to the array returned by the Array property are made to the original array. If this is undesirable, you should perform operations on a copy of the array, rather than an ArraySegment object that represents a portion of the array.

TheEquals(ArraySegment) method and the equality and inequality operators test for reference equality when they compare two ArraySegment objects. For two ArraySegment objects to be considered equal, they must meet all of the following conditions:

  • Reference the same array.

  • Begin at the same index in the array.

  • Have the same number of elements.

If you want to retrieve an element by its index in the ArraySegment object, you must cast it to an IList object and retrieve it or modify it by using the Item[Int32] property. The following example retrieves the element in an ArraySegment object that delimits a section of a string array.

Initializes a new instance of the ArraySegment structure that delimits all the elements in the specified array.

Remarks

This constructor creates an ArraySegment that delimits all the elements of array. That is, the Offset property of the ArraySegment is 0 and its Count property is the length of array. To create an ArraySegment that delimits only part of an array, use the System.ArraySegment`1. constructor.

The original array must be one-dimensional and must have zero-based indexing.

Multiple ArraySegment instances can refer to the same original array and can overlap.
