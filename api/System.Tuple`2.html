Remarks

A tuple is a data structure that has a specific number and sequence of values. The Tuple class represents a 2-tuple, or pair, which is a tuple that has two components. A 2-tuple is similar to a KeyValuePair structure.

You can instantiate a Tuple object by calling either the System.Tuple`2. constructor or the static Create(T1, T2) method. You can retrieve the values of the tuple's components by using the read-only Item1 and Item2 instance properties.

Tuples are commonly used in four different ways:

  • To represent a single set of data. For example, a tuple can represent a record in a database, and its components can represent that record's fields.

  • To provide easy access to, and manipulation of, a data set. The following example defines an array of Tuple objects that contain the names of students and their corresponding test scores. It then iterates the array to calculate the mean test score.

  • Its two components are equal to those of the current instance. Equality is determined by the default object equality comparer for each component.

IStructuralComparable.CompareTo(Object, IComparer)

Compares the current Tuple object to a specified object by using a specified comparer, and returns an integer that indicates whether the current object is before, after, or in the same position as the specified object in the sort order.

Remarks

This member is an explicit interface member implementation. It can be used only when the Tuple instance is cast to an IStructuralComparable interface.

Although this method can be called directly, it is most commonly called by collection sorting methods that include IComparer parameters to order the members of a collection., For example, it is called by the Sort(Array, IComparer) method and the Add(Object, Object) method of a SortedList object that is instantiated by using the System.Collections.SortedList. constructor.

The System.Tuple`2.System method is intended for use in sorting operations. It should not be used when the primary purpose of a comparison is to determine whether two objects are equal. To determine whether two objects are equal, call the System.Tuple`2.System method.

IStructuralEquatable.Equals(Object, IEqualityComparer)

Returns a value that indicates whether the current Tuple object is equal to a specified object based on a specified comparison method.

Remarks

This member is an explicit interface member implementation. It can be used only when the Tuple instance is cast to an IStructuralEquatable interface.

The System.Tuple`2.System implementation is called only if other is not null, and if it can be successfully cast (in C#) or converted (in Visual Basic) to a Tuple object whose components are of the same types as the current instance. The System.Tuple`2.System method first passes the Item1 values of the Tuple objects to be compared to the Equals(Object, Object) implementation. If this method call returns true, the method is called again and passed the Item2 values of the two Tuple objects.

IComparable.CompareTo(Object)

Compares the current Tuple object to a specified object and returns an integer that indicates whether the current object is before, after, or in the same position as the specified object in the sort order.

Remarks

This member is an explicit interface member implementation. It can be used only when the Tuple instance is cast to an IComparable interface.

This method provides the CompareTo(Object) implementation for the Tuple class. Although the method can be called directly, it is most commonly called by the default overloads of collection sorting methods, such as Sort(Array) and Add(Object, Object), to order the members of a collection.

The System.Tuple`2.System method is intended for use in sorting operations. It should not be used when the primary purpose of a comparison is to determine whether two objects are equal. To determine whether two objects are equal, call the Equals(Object) method.

The System.Tuple`2.System method uses the default object comparer to compare each component.
