Delegate WaitOrTimerCallback

Represents a method to be called when a WaitHandle is signaled or times out.

Namespace:System.Threading
Assembly:System.Threading.ThreadPool.dll
Syntax
public delegate void WaitOrTimerCallback(object state, bool timedOut);
Remarks

WaitOrTimerCallback represents a callback method that you want to execute when a registered wait handle times out or is signaled. Create the delegate by passing your callback method to the WaitOrTimerCallback constructor. Your method must have the signature shown here.

Create the registered wait handle by passing the WaitOrTimerCallback delegate and a WaitHandle to RegisterWaitForSingleObject(WaitHandle, WaitOrTimerCallback, Object, UInt32, Boolean). Your callback method executes each time the WaitHandle times out or is signaled.

[Visual Basic]

Visual Basic users can omit the WaitOrTimerCallback constructor, and simply use the AddressOf operator when passing the callback method to RegisterWaitForSingleObject(WaitHandle, WaitOrTimerCallback, Object, UInt32, Boolean). Visual Basic automatically calls the correct delegate constructor.

If you want to pass information to your callback method, create an object that contains the necessary information and pass it to RegisterWaitForSingleObject(WaitHandle, WaitOrTimerCallback, Object, UInt32, Boolean) when you create the registered wait handle. Each time your callback method executes, the state parameter contains this object.

For more information about using callback methods to synchronize thread pool threads, see The Managed Thread Pool.
