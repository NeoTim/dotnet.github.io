WaitAll(WaitHandle[])

Waits for all the elements in the specified array to receive a signal.

Remarks

AbandonedMutexException is new in the .NET Framework version 2.0. In previous versions, the method returns true when a mutex is abandoned. An abandoned mutex often indicates a serious coding error. In the case of a system-wide mutex, it might indicate that an application has been terminated abruptly (for example, by using Windows Task Manager). The exception contains information useful for debugging.

The method returns when all the handles are signaled. On some implementations, if more than 64 handles are passed, a NotSupportedException is thrown. If the array contains duplicates, the call fails with a System.DuplicateWaitObjectException.

The method is not supported on threads that have STAThreadAttribute.

Calling this method overload is equivalent to calling the WaitAll(WaitHandle[], Int32, Boolean) method overload and specifying -1 (or Infinite) for millisecondsTimeoutand true for exitContext.

WaitAll(WaitHandle[], Int32)

Waits for all the elements in the specified array to receive a signal, using an Int32 value to specify the time interval.

Remarks

If millisecondsTimeout is zero, the method does not block. It tests the state of the wait handles and returns immediately.

The method returns when the wait terminates, which means either when all the handles are signaled or when time-out occurs. On some implementations, if more than 64 handles are passed, a NotSupportedException is thrown. If there are duplicates in the array, the call fails with a System.DuplicateWaitObjectException.

The method is not supported on threads that have STAThreadAttribute.

Calling this method overload is the same as calling the WaitAll(WaitHandle[], Int32, Boolean) overload and specifying false for exitContext.

WaitAny(WaitHandle[])

Waits for any of the elements in the specified array to receive a signal.

Remarks

AbandonedMutexException is new in the .NET Framework version 2.0. In previous versions, the method returns true if the wait completes because a mutex is abandoned. An abandoned mutex often indicates a serious coding error. In the case of a system-wide mutex, it might indicate that an application has been terminated abruptly (for example, by using Windows Task Manager). The exception contains information useful for debugging.

The method throws an AbandonedMutexException only when the wait completes because of an abandoned mutex. If waitHandles contains a released mutex with a lower index number than the abandoned mutex, the method completes normally and the exception is not thrown.

In versions of the .NET Framework earlier than version 2.0, if a thread exits or aborts without explicitly releasing a Mutex, and that Mutex is at index 0 (zero) in a WaitAny array on another thread, the index returned by WaitAny is 128 instead of 0.

This method returns when any handle is signaled. If more than one object becomes signaled during the call, the return value is the array index of the signaled object with the smallest index value of all the signaled objects. On some implementations, if more that 64 handles are passed, a NotSupportedException is thrown.

Calling this method overload is equivalent to calling the WaitAny(WaitHandle[], Int32, Boolean) method overload and specifying -1 (or Infinite) for millisecondsTimeoutand true for exitContext.

WaitOne()

Blocks the current thread until the current WaitHandle receives a signal.

Remarks

AbandonedMutexException is new in the .NET Framework version 2.0. In previous versions, the method returns true when a mutex is abandoned. An abandoned mutex often indicates a serious coding error. In the case of a system-wide mutex, it might indicate that an application has been terminated abruptly (for example, by using Windows Task Manager). The exception contains information useful for debugging.

The caller of this method blocks indefinitely until the current instance receives a signal. Use this method to block until a WaitHandle receives a signal from another thread, such as is generated when an asynchronous operation completes. For more information, see the IAsyncResult interface.

Calling this method overload is equivalent to calling the WaitOne(Int32, Boolean) method overload and specifying -1 or Infinite for the first parameter and false for the second parameter.
