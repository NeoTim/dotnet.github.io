Class Tuple<T1>

Represents a 1-tuple, or singleton.

public class Tuple<T1> : IStructuralComparable, IStructuralEquatable, IComparable
Remarks

A tuple is a data structure that has a specific number and sequence of values. The Tuple class represents a 1-tuple, or singleton, which is a tuple that has a single component. A singleton is used comparatively rarely in application development.

You can instantiate a Tuple object by calling either the System.Tuple`1. constructor or the static Create(T1) method. You can retrieve the value of the tuple's single component by using the read-only Item1 instance property.

  • Its single component is equal to that of the current instance. Equality is determined by the default object equality comparer for each component.

IStructuralComparable.CompareTo(Object, IComparer)

Compares the current Tuple object to a specified object by using a specified comparer, and returns an integer that indicates whether the current object is before, after, or in the same position as the specified object in the sort order.

Remarks

Although this method can be called directly, it is most commonly called by collection sorting methods that include IComparer parameters to order the members of a collection. For example, it is called by the Sort(Array, IComparer) method and the Add(Object, Object) method of a SortedList object that is instantiated by using the System.Collections.SortedList. constructor.

The System.Tuple`1.System method is intended for use in sorting operations. It should not be used when the primary purpose of a comparison is to determine whether two objects are equal. To determine whether two objects are equal, call the System.Tuple`1.System method.

IStructuralEquatable.Equals(Object, IEqualityComparer)

Returns a value that indicates whether the current Tuple object is equal to a specified object based on a specified comparison method.

Remarks

This member is an explicit interface member implementation. It can be used only when the Tuple instance is cast to an IStructuralEquatable interface.

The Equals(Object, Object) implementation is called only if other is not null, and if it can be successfully cast (in C#) or converted (in Visual Basic) to a Tuple object whose single component is of the same type as the current instance. The method is passed the Item1 component of the current instance and the Item1 component of the Tuple object represented by the other parameter.

IComparable.CompareTo(Object)

Compares the current Tuple object to a specified object, and returns an integer that indicates whether the current object is before, after, or in the same position as the specified object in the sort order.

Remarks

This member is an explicit interface member implementation. It can be used only when the Tuple instance is cast to an IComparable interface.

This method provides the CompareTo(Object) implementation for the Tuple class. Although the method can be called directly, it is most commonly called by the default overloads of collection sorting methods, such as Sort(Array) and Add(Object, Object), to order the members of a collection.

The System.Tuple`1.System method is intended for use in sorting operations. It should not be used when the primary purpose of a comparison is to determine whether two objects are equal. To determine whether two objects are equal, call the Equals(Object) method.

The System.Tuple`1.System method uses the default object comparer.
